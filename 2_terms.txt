
Top-level object:
 . root object for all the other objects in a Vulkan project; a Vulkan instance
 . "In **Vulkan**, a **“top-level object”** means the first object your program must create (the **instance**)
    because all other Vulkan objects—like devices, surfaces, and queues—are created through or depend on it."

Vulkan device:
 . "In **Vulkan**, a **device** refers to a GPU available to the system—either as a **physical device** (the
    actual graphics hardware) or a **logical device** (the software interface your program uses to submit work to that GPU)."

LLVM pipe:
 . "**LLVMpipe** is a CPU-based software renderer in the **Mesa** graphics stack that uses **LLVM** to compile
    shader code so graphics APIs like **OpenGL** can run even when no GPU acceleration is available."

GPU command queue:
 . "A **GPU command queue** in **Vulkan** is a FIFO stream where a program submits recorded GPU commands (such
    as drawing, memory transfers, or compute work) for the GPU to execute asynchronously."

GPU command queue family:
 . a set of queues for graphics ops
 . "In **Vulkan**, a **graphics-capable queue family** is a group of GPU command queues that support graphics
    operations (such as drawing and rendering) indicated by the `VK_QUEUE_GRAPHICS_BIT` capability flag."

Command pool:
 . manages memory meant for command buffers

Command buffer:
 . a sequence of GPU instructions cached and moved to a GPU queue for execution
 => The pool loads up the buffer with GPU ops and moves them to a command queue for the GPU.
